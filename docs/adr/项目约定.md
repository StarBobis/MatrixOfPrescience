# 核心思想
- 阅读代码优先使用CodeGraph，注意及时更新防止内容过期。
- 模块化代码，前端功能要按组件拆分独立.ts, .vue防止每文件行数超过1000行
- 代码和注释要写纯英文
- 改代码别忘了同步改i18n注意编码问题别导致出现一堆乱码，如果出现及时修复
- 每次修改完代码后基于git diff给我一段简短的中文描述，方便我提交代码使用
- 本项目使用bun,vite作为前端包管理器，不要裸用npm
- 后端应该模块化，可复用内容尽可能使用struct + imple做成工具类
- 不符合核心思想的给我报告风险点

# Rust代码规范

- mod.rs中不允许出现除了pub 声明之外的其它实际代码块，实际代码块应该分模块化各自位于自己独立的.rs文件中，由mod.rs负责向外暴漏
- lib.rs应该尽量维持简洁，功能代码应该模块化放到各个子文件夹下对应rs中而不是堆在一起放在lib.rs中
